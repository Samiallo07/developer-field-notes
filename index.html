Commands, habits, and reminders for my bootcamp journey.

Git vs GitHub

Git is a local version control tool that tracks changes on my computer. GitHub is an online platform where I can store my Git repositories, share projects, and collaborate with others.

Daily Git Loop

  1. Edit my files
  2. Check changes with git status and git diff
  3. Stage files with git add
  4. Commit with a clear message
  5. Push to GitHub

Git Command Reference

  • git init — starts a new Git repository.
  • git status — shows changed and untracked files.
  • git add — stages files for the next commit.
  • git commit -m — saves a snapshot with a message.
  • git diff — shows exact changes before committing.
  • git log --oneline — shows short commit history.
  • git branch — lists branches.
  • git push — uploads commits to GitHub.

Things I Should Not Commit

I should not commit .env files, secrets, passwords, API keys, node_modules, or .DS_Store. These files can expose private information or make the repository messy.
